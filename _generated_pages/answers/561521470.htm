像 Swift 和 Kotlin 這種語法風格叫什麽，和 Java 比有哪些優勢？

2018-12-28 05:30
答案

Dim a as string ="5"

Var a: string ="5"

String a ="5"

我們都一致認為5 放在後面的，因為5是可選的。

但到int32 呢？

Dim a as int32

Var a : i32

時i32 也是可能被後文推導的，

所以也是可選的。

Var a 和 不寫 就只餘下可讀性問題了。

在今天大部分人還是認為變數要宣告的氣氛下，Var a : type = x 擁有最高的自由度。

另外，在fp 或者immutable 中，把型態前置也浪費很多關鍵字空間。

例如ops a b時，ops 就必須用大小寫或者其他方法來區分方法和自定義類型了。
