使用微軟歷代 Windows 版本是怎樣的一番體驗？

2020-02-19 11:47
答案

我雖然現在幾乎只在*nix 上開發。但我實際是個軟粉。

Win1.0 生產力太小，因為解像度規限了。最深刻的是在這麼早的時期就開始了tiling windows manager。到今天為止都是鍵盤派的理想環境。為人津津樂道的i3的理念。三十幾年前就有了。

Win3.1 : 真正的多工環境。生產力上昇無限倍。

Windows95/osr: 神一樣的半擬物ui ，幾乎無法超越的空間效率和辨識度的平衡。加入ie4 後，把url 理念帶進檔案管理器，統一化資源路徑這個概念。後有directX directsound 等底層遊戲接口，欣欣向榮。

Win98/SE: 內置了高版本IE，強化了很多的系統組件和設定。完全令人類意識到計算機的功用是甚麼。

Windows Workstation 3/4: 有完善的建設網絡功能，卻沒有服務器。平價版的NT 內核視窗。

Windows 2000: 萬能的服務器，開展IIS 的插件化。

Windows ME: 基礎上和windows 98 沒分別，加入了視窗主題和active desktop 等華而不實的東西。好處是內置的驅動十分齊全，大量的generic device 都是可用的。而且加入熱重啓，重新開機的速度史無前例。

Windows code name Longhorn: XP 的前身，也是我試過最多個release 的開發版。因為使用NT 核心，就算是開發版也比ME 的穩定性強上很多。主題十分有時代感，也開始加入不單調的操作動畫。在爭紮後移除了active desktop 轉為加入一個很好看的sidebar, 後來轉為xp 的windows widgets.

Windows XP：放棄了本來的ui 而改成有塑料感的漸層色。加入了為win9x 和dos 而大幅暫時改變os 環境的兼容模式，而且兼容模式可以按程序分別設置並同時執行。在專業版也備有IIS 和組織化網絡設定。是個包羅萬有的版本。

Windows Server 2003: 對我衝擊力很大的一個版本。除了一般http 和ftp 等傳統的IIS 功能，還內置有pop ，smtp， 影片串流，音頻串流，等等幾乎能想到的當時能在互聯網上用到的服務器。而且也可以如windows xp 一樣地作日常用途。

Windows server 2008, 2012等，windows vista, windows 7, windows 10 等大家都熟悉的就不多說了。
